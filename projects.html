Programmable Logic Device

February 25th, 2020 to March 2nd, 2020

Digital Electronics

For this project I connected a seven segment display to a programmable logic device. The PLD would output signals to the SSD so that a message could be displayed. I programmed the PLD so that the circuit would display the message, "Secret to everyone". Logic expressions were created so that each segment would light up at the correct states, forming the characters used in the phrase. Using a PLD is much faster than creating the circuit using integrated circuits with AOI Logic. Moreover, it is quick and easy to edit the logic of the PLD because it does not require rewiring physical components.

Printer AOI Logic

November 20th, 2019 to December 6th, 2019

Digital Electronics

We created a model of the logic expression a printer would use to warn its user when it needs maintenance (eg. refill ink). For this we decided a warning would be raised if there is no paper in the printer, no black ink, or (if the printer is set to print in color) no cyan, magenta, or yellow ink. With this in mind we created a truth table outlining the resulting output of every combination of inputs. From this table I created an unsimplified logic expression using the maxterms, then simplified the expression. The purpose of doing so is to reduce the amount of AOI logic components required to make the printer's warning system. Using a CDS I modeled the circuit to guarantee that it functions the way we intended. Once we confirmed the logic expression was correct, the circuit was created on a breadboard to demonstrate concretely how our design works.

Machine Control

April 19, 2018 to May 4, 2018

Principles of Engineering

In this project our team's goal was to create the software for a residential elevator servicing three floors, as well as to make a design for the elevator and model it using VEX parts and its programming language RobotC. Our elevator design moves the cab using sprockets and chains (the cab is attatched to the chain), and has touch sensors at each floor that allows the system to know which floor the cab is at and when it arrives at a floor. Additionally, there was a button on each floor to call the cab to their respective floor. The software was designed so that whenever a request for the cab to move to a different floor was made via the buttons, the program would detect which floor the cab is on to determine which direction the cab should move, and once moving, it will continue until the sensor on the requested floor detects the cab. If five seconds pass without any floors being called the cab moves down to the first floor. Because my goal was the software behind our elevator my main contribution was the creation of the program as well as its troubleshooting.

Hangman

February 9, 2017 to February 15, 2017

Computer Science and Engineering

The objective of this project was to use the coding language python to make a project that emulated the original game hangman. The game was to use a preset list of strings, and let the user guess the letters in the word the function randomly picked. The players ability to guess had to account as many different keyboard inputs as possible, such as punctuation, different casings (ex. A versus a), and inputting multiple characters at once, which the code was able to account for with different functions that we had learned to use. We had decided, and implemented, our code would treat letters of different casing the same, would ignore punctuation, and treat simultaneous inputs the same as several individual inputs.
